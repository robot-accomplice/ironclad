# v0.8.5 Release Notes

**Date:** 2026-02-28
**Scope:** 50-finding stabilization audit — security hardening, resource exhaustion prevention, silent failure elimination

## Highlights

This release addresses all 50 findings from the v0.8.5 bug audit across 30 files in 9 workspace crates.
The primary themes are:

1. **Preemptive resource protection** — WASM and script execution now have true preemptive timeouts that kill runaway processes, rate limiter maps are bounded to prevent memory exhaustion, and file reads are capped at safe limits.

2. **Silent failure elimination** — 38 sites across the codebase where errors were silently discarded via `let _ =`, `.unwrap_or_default()`, or `.ok()` now surface diagnostics through structured logging.

3. **Panic path removal** — Signal handler setup, reqwest client initialization, heartbeat tier matching, and regex compilation no longer use `.expect()` or `unreachable!()` in paths reachable at runtime.
