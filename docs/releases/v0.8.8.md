# v0.8.8 Release Notes

Security audit release closing 37 of 38 findings from the comprehensive v0.8.8 security audit — 8 HIGH, 17 MED, 12 LOW across 7 batches. One HIGH (WebSocket ticket auth) deferred to a design PR.

## Highlights

- **HIGH: WebSocket API key leak**: Replaced `?token=` query-string auth with a ticket-based flow, preventing API keys from appearing in server/proxy logs and browser history
- **HIGH: Prompt injection in tips**: LLM-generated tips are now sanitized before rendering, closing a stored prompt injection vector
- **HIGH: Tool call parsing**: `parse_tool_call` now handles delimiter characters in arguments via `rfind` with a candidate loop
- **HIGH: P50 latency**: True median calculation for even-length arrays
- **MED: XSS defense**: `sanitize_html` now escapes all 5 OWASP-recommended HTML entities
- **12 silent failure fixes**: Replaced `.ok()`, `let _ =`, and empty error bodies with proper error propagation and logging
