# v0.8.1 Release Gate Checklist (Stabilization)

This is a stabilization release resolving 40 smoke/UAT bugs found during comprehensive testing of v0.8.0.

## Scope

- 40 bug fixes across API, CLI, scheduler, dashboard, and memory subsystems
- CI script portability improvements (POSIX grep compatibility)
- Release pipeline cleanup (remove stale ripgrep dependency)

## Bug Fix Summary

| Severity | Count | Areas |
|----------|-------|-------|
| Critical/Security | 5 | Input validation, HTML injection, path exposure |
| High | 6 | JSON error format, session response, 404 handler, rate limiter |
| Medium | 15 | Memory search, cron scheduler, cost precision, skills metadata, CLI resilience |
| Low/UX | 14 | Dashboard display, pagination, status indicators, font dependency |

## Release Gates

### Input Validation & Security

- [x] Field-length validation on all POST bodies (agent_id/name capped at 256, descriptions at 4096).
- [x] HTML/script injection prevention via sanitization at write boundaries.
- [x] Null-byte rejection in string fields.
- [x] Breaker reset validates provider existence before returning success.
- [x] Skills API no longer exposes filesystem paths.

### API Error Format

- [x] All error responses return structured `{"error": "..."}` JSON.
- [x] `POST /api/sessions` returns full session object.
- [x] Unknown routes return JSON 404 via fallback handler.
- [x] Rate limiter returns JSON error format.

### Memory & Scheduler

- [x] Memory FTS search deduplicates results and returns structured entries.
- [x] Memory timestamps normalized to UTC RFC3339.
- [x] `classify_turn()` false-positive rate reduced for financial classification.
- [x] Cron `next_run_at` persisted after computation.
- [x] Heartbeat no longer floods logs with virtual job IDs.
- [x] Cron jobs use actual agent IDs.

### CLI & Dashboard

- [x] `ironclad check` shows friendly error with config path suggestion.
- [x] CLI `status` shows "0%" instead of "n/a" for zero-hit cache.
- [x] Dashboard: 14 UX fixes including pagination, status display, uptime formatting, external font removal.

### CI & Release Pipeline

- [x] All CI scripts use POSIX `grep` instead of `rg` (ripgrep).
- [x] Release workflow (`release.yml`) no longer installs ripgrep.
- [x] Windows compilation: conditional `allow(unused_mut)` for platform-gated code.

### Zero-Regression Gate

- [x] `just ci-test` passes all 18 stages at 88.02% coverage.
- [x] GitHub CI passes all 28 checks on PR #39.
- [x] Cross-platform: macOS, Ubuntu, and Windows tests pass.
