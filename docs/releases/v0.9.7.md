# v0.9.7 Release Checklist — DB Fitness, Hardening & v1.0 Prep

Ironclad runs on an embedded SQLite database — a mal-performant data layer sinks the entire ship. The v0.9.6 DB health audit identified 18 concrete issues across 4 severity tiers. This release resolves all of them, adds operational hardening, and evaluates v1.0 readiness.

## Scope

Primary:

- **DB Fitness items 1–18** — SQLite performance hardening across all severity tiers
- **Sandbox Boundary Management** — Hardened execution boundaries for skill scripts
- **Memory Hygiene Mechanic** — Self-service detection and purge of contaminated memory entries
- **Unified Pipeline Architecture** — IntentRegistry, GuardChain, ShortcutDispatcher, PipelineConfig

### Operational Fixes (Live-Testing Findings)

- [x] **Circuit breaker window reset**: `record_failure()` now tracks `window_start` for rolling-window accumulation — failures spaced ~60s apart correctly accumulate to threshold instead of resetting
- [x] **Embedding auth for local providers**: `EmbeddingConfig.is_local` skips API key resolution and auth headers for Ollama/llama.cpp — fixes "OLLAMA-GPU_API_KEY not found" failures
- [x] **Cron `schedule_kind: "once"` support**: Runtime now maps "once" → "at" dispatch alias, calls `DurableScheduler::evaluate_at()`, auto-disables job after single execution — fixes 60s warning spam
- [x] **Vault path whitelisting**: `tool_allowed_paths` field on `FilesystemSecurityConfig`, auto-populated from `obsidian.vault_path` during config normalization — `PathProtectionRule` checks whitelist before denying absolute paths in workspace-only mode
- [x] **Fleet activity chart capacity model**: Stacked area chart normalizes per-agent scores by `1/agentCount` with `fixedMax: 1.0` — 1 of 14 agents active shows ~7% not 100%
- [x] **Wallet RPC rate-limit backoff**: `get_native_balance()` and `get_erc20_balance()` detect rate-limit codes (`-32016`, `-32005`, `429`); `get_all_balances()` breaks token iteration on first rate-limit hit
- [x] **Cron once-type orphan auto-disable**: Jobs with `schedule_kind: "once"` and no `schedule_expr` now auto-disabled on first evaluation instead of warning every 60s
- [x] **Dashboard sidebar footer pinning**: Added `height: 100%` to `.sidebar` CSS — footer stays viewport-locked
- [x] **Dashboard custom model Add button**: Custom text-input row now has its own Add button; event handler uses shared `.roster-model-add-btn` class selector
- [x] **Clippy lint cleanup**: Removed dead `KeyStatus::OAuth` variant, fixed 5 `collapsible_if` → `if let` chains, 1 `let_and_return` fix, `#[allow(dead_code)]` on forward-looking pipeline scaffolding
