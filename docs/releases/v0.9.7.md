# v0.9.7 Release Checklist — DB Fitness, Hardening & v1.0 Prep

Ironclad runs on an embedded SQLite database — a mal-performant data layer sinks the entire ship. The v0.9.6 DB health audit identified 18 concrete issues across 4 severity tiers. This release resolves all of them, adds operational hardening, and evaluates v1.0 readiness.

## Scope

Primary:

- **DB Fitness items 1–18** — SQLite performance hardening across all severity tiers
- **Sandbox Boundary Management** — Hardened execution boundaries for skill scripts
- **Memory Hygiene Mechanic** — Self-service detection and purge of contaminated memory entries

Feature delivery (parallel or follow-on):

- `3.1` Compile-Time Safety (typestates design)
- `2.16` Matrix Channel (E2EE)
- `3.9` Storage Backend Trait

## Release Gates

### DB Fitness — CRITICAL (unbounded growth, data loss risk)

- [ ] **DF-1**: Unbounded high-write table pruning — configurable retention + pruning for `turns`, `session_messages`, `tool_calls`, `inference_costs`, `model_selection_events`
- [ ] **DF-2**: Working-memory orphan cleanup — `working_memory` rows reference expired sessions with no FK cascade; add ON DELETE CASCADE or periodic sweep
- [ ] **DF-3**: Embedding orphan cleanup — `embeddings` table has no FK cascade from parent memory tables; orphaned BLOBs accumulate after memory pruning
- [ ] **DF-4**: VACUUM strategy — `auto_vacuum=NONE` means deleted rows waste space forever; implement `PRAGMA incremental_vacuum` or switch to `auto_vacuum=INCREMENTAL`

### DB Fitness — HIGH (missing indexes, query inefficiency)

- [ ] **DF-5**: Index `turns.session_id` — high-frequency query path with no index; full table scan on every turn lookup
- [ ] **DF-6**: Index `working_memory.session_id` — queried on every context build, no index
- [ ] **DF-7**: Fix N+1 in `rerank_episodic_by_decay` — currently loads all rows then filters in Rust; rewrite as single SQL query with decay math
- [ ] **DF-8**: Episodic memory dead-entry cleanup — entries decay to `importance <= 1` but are never deleted; add threshold-based pruning to hygiene sweep

### DB Fitness — MEDIUM (silent waste, stale infrastructure)

- [ ] **DF-9**: `cron_runs` pruning + index — unbounded with no index on `job_id`; add retention policy and covering index
- [ ] **DF-10**: `policy_decisions` indexes — no indexes at all; add indexes on `session_id` and `created_at`
- [ ] **DF-11**: `context_snapshots` pruning — unbounded; add configurable retention (keep N per session)
- [ ] **DF-12**: `shadow_routing_predictions` pruning — unbounded; add time-based retention
- [ ] **DF-13**: Wire `evict_expired_cache()` — function exists but is never called; wire into governor tick or cache flush cycle
- [ ] **DF-14**: Fix `semantic_cache` NULL expiry — rows with `expires_at IS NULL` are never evicted; add default expiry or handle in eviction

### DB Fitness — LOW (normalization, tuning, dead code)

- [ ] **DF-15**: Remove dual `embedding_json`/`embedding_blob` — migration artifact; all reads use BLOB path; drop JSON column and migration shim
- [ ] **DF-16**: Normalize low-cardinality TEXT columns — 25+ columns store repeated strings (`status`, `tier`, `memory_type`, etc.); evaluate enum tables or CHECK constraints
- [ ] **DF-17**: PRAGMA tuning under WAL — `synchronous=FULL` under WAL wastes ~2x write throughput; switch to `synchronous=NORMAL` (safe under WAL)
- [ ] **DF-18**: Remove `proxy_stats` dead table — never written to; drop table and migration

### Sandbox Boundary Management

- [ ] Filesystem confinement: skill scripts restricted to `skills_dir` subtree and explicit `$IRONCLAD_WORKSPACE` (no `..` traversal, no symlink escape)

- [ ] Resource limits: wall-clock timeout enforcement hardened (existing `script_timeout_seconds`), add memory ceiling via `ulimit`/`rlimit`
- [ ] Interpreter allowlist audit: validate `allowed_interpreters` against absolute paths to prevent PATH hijacking
- [ ] Mechanic check: `ironclad mechanic` reports sandbox configuration health (permissive interpreters, disabled sandbox, missing limits)

## Implementation Order

1. **DF-17** (PRAGMA tuning) — one-line change, immediate throughput gain
2. **DF-5, DF-6, DF-9, DF-10** (missing indexes) — CREATE INDEX migrations, no code changes
3. **DF-4** (VACUUM strategy) — PRAGMA change + periodic incremental vacuum wiring
4. **DF-18** (dead table removal) — DROP TABLE migration
5. **DF-15** (dual embedding column) — DROP COLUMN migration + code cleanup
6. **DF-13, DF-14** (cache wiring) — wire existing function + fix NULL expiry
7. **DF-2, DF-3** (orphan cleanup) — FK cascade or periodic sweep logic
8. **DF-7** (N+1 fix) — rewrite rerank query
9. **DF-1, DF-8, DF-11, DF-12** (retention/pruning) — configurable pruning with governor integration
10. **DF-16** (normalization) — evaluate and apply CHECK constraints where beneficial
11. Sandbox boundary management
12. Feature items (3.1, 2.16, 3.9) in parallel
