# v0.9.5 Release Checklist (Autonomy Infrastructure)

This release follows `v0.9.4` and focuses on reliable autonomous task execution.

Deferred capability governance follows
`docs/architecture/deferred-capability-policy.md`.

## Scope

- `2.4` Speculative Execution
- `1.19` Browser External Runtime
- `1.24` CLI Agent Skills (Claude Code + Codex)
- `2.18` Revenue Autonomy Scaffold + Phase 1 paid-service loop
- Duncan behavior polish cycle (response quality + continuity hardening)
- Revenue Phase 2 shared control-plane primitives (intake + qualification gate)

## Execution Order

### Phase 0: Safety & Control Plane

0a. **Autonomy policy envelope**

- Define explicit policy boundaries for speculative tool execution and external runtimes.
- Ensure all autonomy paths emit auditable policy decisions.

0b. **Execution budget guards**

- Add hard limits for parallel branches, runtime duration, and external tool budget.
- Ensure deterministic cancel/cleanup semantics.

### Phase 1: `2.4` Speculative Execution

1a. **Branch planner + scorer integration**

- Enable multi-branch candidate execution for high-latency turns.
- Score branches and commit best result under policy and budget constraints.

1b. **Branch cancellation and result safety**

- Cancel losing branches cleanly.
- Ensure side-effecting tools are never speculatively executed without explicit safe mode.

### Phase 2: `1.19` Browser External Runtime

2a. **External runtime abstraction**

- Implement runtime boundary for browser operations outside core process.
- Add lifecycle management, health checks, and failure recovery.

2b. **Security and sandbox controls**

- Enforce URL/domain, file access, and capability restrictions.
- Add operator-facing diagnostics for blocked/allowed actions.

### Phase 3: `1.24` CLI Agent Skills

3a. **CLI skill adapter interfaces**

- Add typed adapters for Claude Code and Codex CLI execution as skills.
- Normalize request/response format and tool metadata.

3b. **Credential and environment isolation**

- Ensure per-skill environment scoping and secret handling.
- Add timeout/retry/error classification for CLI-backed skills.

### Phase 4: Integration & Operationalization

4a. **Cross-feature integration tests**

- Validate speculative + browser + CLI skill interactions under budget/policy pressure.

4b. **Operator controls**

- Add enable/disable toggles and diagnostics for each autonomy subsystem.

### Phase 5: `2.18` Revenue Autonomy Scaffold

5a. **Control-plane scaffold**

- Implement one canonical lifecycle for revenue work (intake, gate, score, plan, fulfill, settle, feedback).
- Ensure all revenue strategies reuse this control plane without bypasses.

5b. **Phase 1 paid-service loop**

- Ship one paid service (`geopolitical-sitrep-verified`) end-to-end.
- Wire quote -> payment verification -> fulfillment -> ledger settlement with persistent DB records.

### Phase 6: Deferred-Capability Governance + Phase 2 Revenue Continuation

6a. **Deferred capability policy operationalization**

- Link release flow to deferred capability policy.
- Record out-of-scope capabilities explicitly to prevent reintroduction drift.
- Keep Conway integration explicitly deferred and non-operational for v0.9.5.

6b. **Revenue Phase 2 control-plane continuation**

- Add opportunity intake + qualification gate primitives reused by all revenue strategies.
- Implement first micro-bounty adapter against shared lifecycle (no bespoke path).
- Add mechanic probe/reconcile/orphan-repair hooks for revenue control-plane health.
