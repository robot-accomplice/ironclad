# v0.9.0 Release Checklist (Smarter, Leaner, Faster)

This release focuses on making the agent smarter and cheaper to run (prompt compression, metascore routing, context checkpoint) while hardening the runtime for production (durable delivery queue) and unlocking external browser backends.

## Scope

Primary:

- `2.8` Prompt Compression
- `1.11` Context Checkpoint
- `1.16` Durable Channel Delivery Queue
- `2.19` Model Metascore Routing Profiles
- `1.19` `agent-browser` External Runtime Support

Stretch:

- `1.18` Cron-Conformant Session Rotation
- `2.12` Episodic Digest System

## Release Gates

### 1.11 Context Checkpoint

- [ ] `checkpoints` table exists in schema with versioned format and migration.
- [ ] `ContextCheckpoint` struct captures compiled system prompt, top-k memory summaries, active task list, and recent conversation digest.
- [ ] Checkpoint is written every N turns (configurable via `[context.checkpoint]`) and on graceful shutdown.
- [ ] `load_checkpoint()` provides instant agent readiness on boot; background retrieval warms full context asynchronously.
- [ ] Crash recovery loses at most N turns of checkpoint state; raw data remains intact in DB.
- [ ] Checkpoint format is versioned; stale/incompatible checkpoints are discarded gracefully.
- [ ] Boot-time latency measurably improves with warm checkpoint vs. cold start.

### 1.16 Durable Channel Delivery Queue

- [ ] Retryable outbound events persisted to DB with attempt count, next-attempt timestamp, and terminal failure reason.
- [ ] Restart recovery replays pending deliveries from persistent storage.
- [ ] Idempotent delivery keys prevent duplicate sends after recovery.
- [ ] Dead-letter queue is inspectable and replayable via API and CLI.
- [ ] Channel health metrics reflect queue depth and retry rates.

### 2.8 Prompt Compression

- [ ] Perplexity-based token importance scoring implemented (local model or primary local model).
- [ ] Tokens below configurable threshold are pruned before inference.
- [ ] Compression targets long conversation histories and large tool descriptions.
- [ ] Gated behind `cache.prompt_compression = true` (off by default).
- [ ] Measurable token reduction (target: 2-20x) with <5% quality degradation on eval suite.
- [ ] Compression metrics visible in Context Observatory efficiency stats.

### 2.19 Model Metascore Routing Profiles

- [ ] `ModelProfile` schema defined per model with weighted attributes: task efficacy, cost, availability/reliability, locality/privacy, resource requirements.
- [ ] `TaskRequirements` schema captures hard constraints (privacy, budget cap, latency target, minimum quality) and soft preferences.
- [ ] `metascore(model, task)` computed from profile x requirements with transparent scoring and deterministic tie-breakers.
- [ ] Hard-constraint gate eliminates ineligible candidates with explicit fail reasons (per `METASCORE_V1_SPEC.md`).
- [ ] Metascore selection integrated into direct routing and multi-model orchestration paths.
- [ ] Profile snapshots and outcomes persisted for weight tuning from observed quality/cost/latency data.
- [ ] Operator controls for policy weights exposed via config and API.
- [ ] Dashboard explainability panel shows per-task match reasoning (criteria scores, constraint passes/fails, tie-break path).
- [ ] Simulation mode: operators can run "what would be selected?" scenarios against alternative requirements/weights.

### 1.19 `agent-browser` External Runtime Support

- [ ] Browser backend abstraction trait introduced (`BrowserBackend`).
- [ ] `agent-browser` backend implementation maps core action set (`open`, `snapshot`, `click`, `fill`, `wait`, `get text`, `screenshot`) to CLI commands with `--json` result parsing.
- [ ] Config for binary path, provider/session/profile defaults, domain allowlist, and output-size limits.
- [ ] Existing safety rails (`RiskLevel`, approvals) preserved for external backend.
- [ ] Provenance events emitted for backend selection and fallback.
- [ ] Integration tests with mock `agent-browser` binary pass.
- [ ] Release smoke checks validate backend availability detection.

### 1.18 Stretch: Cron-Conformant Session Rotation

- [ ] Reset schedules evaluated using configured cron expression directly.
- [ ] Edge schedule and timezone conformance tests pass.
- [ ] Migration notes documented for any behavior change from legacy timing semantics.

### 2.12 Stretch: Episodic Digest System

- [ ] `EpisodicDigest` generated at session boundaries (close, compaction, TTL expiry).
- [ ] Digest captures agent state, key decisions, unresolved tasks, and learned facts.
- [ ] Stored in `episodic_memory` table with `digest` flag and elevated retrieval priority.
- [ ] `MemoryRetriever` automatically surfaces most recent digest as high-priority context on session start.
- [ ] Decay-weighted relevance: recent digests have maximum relevance, older digests fade unless query-matched via hybrid search.
