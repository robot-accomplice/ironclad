# v0.9.0 Release Checklist (Foundation)

This release lays the architectural foundation for the v0.9.x series: instant boot (checkpoint), crash-safe delivery (durable queue), memory continuity (episodic digest), token efficiency (prompt compression), agent self-awareness (introspection), and codebase simplification (single-pathway consolidation). Together these unblock every subsequent v0.9.x milestone.

## Scope

Primary:

- `NEW` Single-Pathway Consolidation
- `1.11` Context Checkpoint
- `1.16` Durable Channel Delivery Queue
- `1.22` Introspection Skill (`get_runtime_context` + `ToolContext` channel extension)
- `2.12` Episodic Digest System
- `2.8` Prompt Compression

Stretch:

- `1.18` Cron-Conformant Session Rotation

Deferred to v0.9.1:

- `2.19` Model Metascore Routing Profiles — too large to share scope with six foundation items; benefits from 2.12's improved context quality and 2.8's compression metrics as inputs.
- `2.3` Tiered Inference Pipeline — routing concern that shares infrastructure with 2.19; co-locating avoids refactoring the routing hot path twice.

Deferred to v0.9.6+:

- `1.19` `agent-browser` External Runtime Support — isolated feature, not on the critical path for infrastructure.

Dropped (superseded):

- `1.23` Context Budget Tuning — the stopgap L0 raise becomes unnecessary once 1.22 provides `get_runtime_context` (info on demand replaces pre-injected bloat).

## Execution Order

1. `NEW` Single-Pathway Consolidation — eliminates the dual-path divergence that caused BUG-3/7/11; every subsequent item benefits from having one path to modify instead of two
2. `1.11` Context Checkpoint — no external dependencies; unblocks faster dev/test cycles for the rest
3. `1.16` Durable Channel Delivery Queue — independent persistence work; can parallel with (2)
4. `1.22` Introspection Skill — small scope (~100 LOC), no dependencies; `ToolContext` extension benefits every subsequent tool implementation
5. `2.12` Episodic Digest System — wires existing `digest_on_close()` into governor; LLM summarization replaces heuristic `from_session()`; decay-weighted retrieval
6. `2.8` Prompt Compression — builds on context assembly pipeline; benefits from checkpoint warm-start and digest-enriched context
7. `1.18` Stretch: Cron-Conformant Session Rotation — small, independent

## Release Gates

### NEW: Single-Pathway Consolidation

- [ ] `agent_message` and `process_channel_message` share a single core processing function.
- [ ] Decomposition gate, `scan_output`, and logging are applied uniformly regardless of entry point (web API, channel, WebSocket).
- [ ] No behavioral divergence between web and channel message paths in test suite.
- [ ] All three turn handlers (REST non-streaming, REST streaming, WS/channel) delegate to the consolidated path.
- [ ] Existing tests pass without modification (behavior-preserving refactor).

### 1.11 Context Checkpoint

- [ ] `checkpoints` table exists in schema with versioned format and migration.
- [ ] `ContextCheckpoint` struct captures compiled system prompt, top-k memory summaries, active task list, and recent conversation digest.
- [ ] Checkpoint is written every N turns (configurable via `[context.checkpoint]`) and on graceful shutdown.
- [ ] `load_checkpoint()` provides instant agent readiness on boot; background retrieval warms full context asynchronously.
- [ ] Crash recovery loses at most N turns of checkpoint state; raw data remains intact in DB.
- [ ] Checkpoint format is versioned; stale/incompatible checkpoints are discarded gracefully.
- [ ] Boot-time latency measurably improves with warm checkpoint vs. cold start.

### 1.16 Durable Channel Delivery Queue

- [ ] Retryable outbound events persisted to DB with attempt count, next-attempt timestamp, and terminal failure reason.
- [ ] Restart recovery replays pending deliveries from persistent storage.
- [ ] Idempotent delivery keys prevent duplicate sends after recovery.
- [ ] Dead-letter queue is inspectable and replayable via API and CLI.
- [ ] Channel health metrics reflect queue depth and retry rates.

### 1.22 Introspection Skill

- [ ] `get_runtime_context` tool returns channel name, session ID, active agent, uptime, and connected channel list — zero cost until called.
- [ ] `get_memory_stats` tool returns memory tier counts, total memories, last retrieval latency.
- [ ] `get_channel_health` tool returns per-channel delivery stats, queue depth, last-seen timestamp.
- [ ] `ToolContext` gains `channel: Option<String>` field — every tool gains channel awareness without system prompt injection.
- [ ] All three tools registered in tool registry with proper JSON schemas.
- [ ] Tools return structured JSON; agent can reason over the output without custom parsing.

### 2.12 Episodic Digest System

- [ ] `digest_on_close()` called from `SessionGovernor` on TTL expiry, rotation, and archive paths.
- [ ] `EpisodicDigest::from_session()` upgraded from heuristic to LLM-based summarization.
- [ ] Digest captures agent state, key decisions, unresolved tasks, and learned facts.
- [ ] Stored in `episodic_memory` table with `digest` flag and elevated retrieval priority.
- [ ] `MemoryRetriever` automatically surfaces most recent digest as high-priority context on session start.
- [ ] Decay-weighted relevance: recent digests have maximum relevance, older digests fade unless query-matched via hybrid search.
- [ ] Resolved items marked with `resolved` flag; retrieval suppresses resolved items unless explicitly queried.

### 2.8 Prompt Compression

- [ ] Perplexity-based token importance scoring implemented (local model or primary local model).
- [ ] Tokens below configurable threshold are pruned before inference.
- [ ] Compression targets long conversation histories and large tool descriptions.
- [ ] Gated behind `cache.prompt_compression = true` (off by default).
- [ ] Measurable token reduction (target: 2-20x) with <5% quality degradation on eval suite.
- [ ] Compression metrics visible in Context Observatory efficiency stats.

### 1.18 Stretch: Cron-Conformant Session Rotation

- [ ] Reset schedules evaluated using configured cron expression directly.
- [ ] Edge schedule and timezone conformance tests pass.
- [ ] Migration notes documented for any behavior change from legacy timing semantics.

## Cut Policy

1. Cut `1.18` (Cron Rotation) first — small scope but not on the critical path.
2. If further reduction is required, defer `1.22` introspection tools (keep `ToolContext` extension only — 10 lines).
3. If further reduction is required, defer `2.12` LLM summarization to a patch release while keeping heuristic `from_session()` wiring and decay-weighted retrieval.
4. Keep `1.11`, `1.16`, `2.8`, and single-pathway consolidation in scope — these form the minimum viable release.
