# v0.9.2 Release Checklist (Operator Visibility & Data Architecture)

This release makes the system observable, operable, and self-aware about its own data. It absorbs the original v0.9.3 (Operator Tooling) and v0.9.4 (Data Architecture) scopes into a single ambitious release.

## Scope

Primary (7 items — 1L + 4M + 2S):

- `2.19` Dashboard panels + explainability — metascore breakdown, candidate comparison, simulation mode
- `1.21` Integrations Management (CLI) — `ironclad integrations list/test/connect/disconnect`
- `2.15` Ops Telemetry + Provenance — release smoke gates, deployment readiness, artifact provenance
- `2.13` Hippocampus — self-describing schema map, agent-created tables with guardrails
- `3.5.5` Document Ingestion Pipeline — PDF/markdown/code → memory system
- `1.18` Cron Rotation — cron-conformant session reset behavior
- `inference_costs` extension — add `latency_ms`, `quality_score`, `escalation` columns

Deferred:

- `1.22` introspection extensions — revisit when v0.9.0/v0.9.1 usage patterns are available

## Execution Order

### Phase 1: Schema & Data Foundation

1. **`inference_costs` schema extension** — Add `latency_ms` (INTEGER), `quality_score` (REAL nullable), `escalation` (BOOLEAN) columns to existing `inference_costs` table. Add migration. Update `persist_inference_cost()` call sites in LlmService to record latency and escalation flag. Back-fill quality_score from QualityTracker when available.

2. **1.18 Cron Rotation** — Evaluate session reset schedules directly from configured cron expressions. Add conformance tests for edge schedules (DST transitions, sub-minute cron, timezone-prefixed expressions). Document migration notes if semantics differ from legacy timing behavior.

3. **Hippocampus table + bootstrap** — Create `hippocampus` table: `table_name TEXT PK`, `owner TEXT` (system | agent_id), `schema_ddl TEXT`, `description TEXT`, `query_patterns TEXT`, `relationships TEXT`, `access_level TEXT` (read_only | read_write | internal), `row_count INTEGER`, `last_modified TEXT`. Populate from all system tables in `initialize_db()`. Add consistency check on boot.

### Phase 2: Agent Data Tools

4. **`CreateTable` / `AlterTable` / `DropTable` tools** — Register in ToolRegistry. Agent-created tables prefixed with agent id (e.g., `ag_duncan_research_notes`). Enforce: no reserved column names, no system table collisions, max tables per agent, max total size. Gate behind `RiskLevel::Caution`. Auto-register in hippocampus.

5. **Hippocampus context injection** — In `build_context()`, inject compact hippocampus summary (table names + one-line descriptions) into system prompt. Budget: ~200 tokens for the summary. Agent gains ambient awareness of its storage capabilities.

6. **3.5.5 Document Ingestion** — CLI: `ironclad ingest <path>` for PDF/markdown/code files. Parse → chunk (512 tokens, 64-token overlap using existing chunking infra) → embed → store in memory system. Support: `.md`, `.txt`, `.rs`, `.py`, `.js`, `.ts`, `.pdf` (via lightweight parser). API: `POST /api/knowledge/ingest` with multipart upload. Register ingested documents in hippocampus as knowledge sources.

### Phase 3: Operator CLI Tooling

7. **1.21 Integrations Management (CLI)** — Implement four subcommands:
   - `ironclad integrations list` — per-channel status summary (enabled, healthy, last error, last message)
   - `ironclad integrations test <channel>` — health probe: token validity, connectivity, permissions
   - `ironclad integrations connect <channel>` — guided setup: prompt for token, store in keystore, write config, run test
   - `ironclad integrations disconnect <channel>` — remove token from keystore, disable in config

8. **Integrations API** — `GET /api/integrations` returning per-channel status. `POST /api/integrations/{channel}/test` for live probes. Wire into existing `ChannelRouter` health metrics.

### Phase 4: Ops Telemetry & Provenance

9. **2.15 Release smoke gates** — Add mandatory pre-tag smoke checks to release pipeline: binary launches, config parses, DB migrates, health endpoint responds. Block `git tag` if any gate fails.

10. **Deployment readiness check** — `ironclad doctor` command: validates config, checks DB connectivity, verifies keystore unlockable, tests provider endpoints, reports overall readiness score.

11. **Artifact provenance** — Emit signed provenance metadata (build hash, commit, timestamp, CI pipeline URL) in release artifacts. Verifiable with `ironclad verify <binary>`.

### Phase 5: Dashboard Explainability

12. **Metascore breakdown panel** — New dashboard tab/section showing per-turn routing decisions: selected model, MetascoreBreakdown (efficacy, cost, availability, locality, confidence, final_score), complexity score, all candidates with individual scores. Source: existing `/api/models/selections` endpoint.

13. **Candidate comparison table** — For each routing decision: all models considered, per-dimension scores, why winner was picked (highest metascore), why others were rejected (breaker blocked, capacity exhausted, score too low). Sortable by any dimension.

14. **Routing simulation mode** — Operator enters task description + optional weight overrides → system runs `select_by_metascore()` against current model profiles without executing inference → returns hypothetical selection with full scoring breakdown. API: `POST /api/models/simulate` with `{ task_description, weight_overrides? }`.

15. **Inference cost trends** — Dashboard visualization of `inference_costs` with new columns: latency distribution, quality trends per model, escalation rates. Time-series charts with model-level drill-down.

## Implementation Notes

### Hippocampus Design Principles

The hippocampus is the agent's "awareness of self-storage". Key constraints:
- Agent-created tables are sandboxed: prefixed, size-limited, no PRAGMA/ATTACH access
- System tables are read-only from the agent's perspective
- The hippocampus summary in context is compact (~200 tokens) — full schema details are available via introspection tools
- Stats (row_count, last_modified) refresh on MetricSnapshot heartbeat, not on every query

### Document Ingestion Architecture

Builds on completed RAG infrastructure (3.5 items 1–4):
- Binary embedding storage (done)
- HNSW ANN index (done)
- Content chunking — 512 tokens, 64-token overlap (done)
- Persistent semantic cache (done)

New work: file parsing → chunking → embedding → storage pipeline. PDF parsing should be lightweight (no heavy C dependencies) — consider `pdf-extract` or similar pure-Rust crate.

### Integrations CLI UX

The `connect` wizard should be channel-aware:
- Telegram: prompts for bot token, validates with getMe API
- WhatsApp: prompts for phone number ID + access token, validates with profile API
- Discord: prompts for bot token, validates with gateway identify
- Slack: prompts for bot token + signing secret, validates with auth.test

Each adapter's `ChannelAdapter` trait already has a health check method — the CLI `test` command calls it directly.

### Simulation Mode

The simulation endpoint reuses `build_model_profiles()` + `select_by_metascore()` but skips actual inference. It runs the full scoring pipeline against a synthetic task with operator-provided complexity hints and weight overrides, returning the same `MetascoreBreakdown` struct the real routing path produces.
