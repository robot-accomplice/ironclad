# v0.8.4 Release Notes

Stabilization release addressing all 20 findings from the v0.8.4 bug audit — 6 security/correctness bugs and 14 silent failure patterns. Also fixes the crate publish ordering issue that blocked v0.8.3's crates.io release.

## Highlights

- **20 bug audit findings resolved**: Systematic elimination of silent failure anti-patterns (`.unwrap_or_default()` on `Result`, `.ok()` on DB writes, `let _ =` on `Result`, `try_lock` with silent drop)
- **3 security hardening fixes**: WebSocket DoS mitigation, TOCTOU race condition, and script runner OOM prevention
- **Crate publish ordering fix**: Release CI now publishes in correct topological dependency order

## Security

| Fix | Severity | Description |
|-----|----------|-------------|
| WebSocket message size limit (BUG-5) | High | 4 KiB inbound limit, no message echo, close on oversize — closes ~3x memory amplification DoS vector. |
| Hippocampus TOCTOU (BUG-4) | Medium | `drop_agent_table` auth check + DROP wrapped in single transaction, preventing race-condition bypasses. |
| Script runner bounded reads (BUG-6) | Medium | Shebang detection uses `BufReader::take(512)` instead of `read_to_string`, preventing OOM on oversized scripts. |

## Bug Fixes — Security & Correctness

| ID | Fix | Description |
|----|-----|-------------|
| BUG-2 | Money::from_dollars NaN panic | Returns `Result` instead of panicking; rejects NaN/Infinity. |
| BUG-3 | Agent loop detection enforcement | `is_looping()` now enforced inside `transition()`, forces `Done` on loop. |
| BUG-7 | Digit-leading SQL identifiers | `validate_identifier` rejects names starting with digits. |

## Bug Fixes — Silent Failures

| ID | Severity | Fix | Description |
|----|----------|-----|-------------|
| SF-1 | Critical | Governor silent write failures | Errors logged at warn/error; accurate expired count returned. |
| SF-2 | Critical | Agent amnesia on DB error | `list_messages` errors propagated instead of empty fallback. |
| SF-4 | Critical | Embedding API key error | Clear error message instead of cryptic 401. |
| SF-3 | High | Admin dashboard false empties | DB errors logged with `inspect_err` before default fallback. |
| SF-5 | High | EventBus publish logging | Debug log on no subscribers instead of silent drop. |
| SF-6 | High | ANN corrupt JSON handling | Corrupt embeddings logged and skipped instead of silently defaulting. |
| SF-7 | High | Delivery queue recovery | Async `.lock().await` replaces `try_lock` that silently dropped messages. |
| SF-8 | High | Dead letter false empties | Errors logged before fallback. |
| SF-9 | High | Session tool call queries | DB errors return HTTP 500 instead of empty arrays. |

| SF-13 | Medium | Efficiency report serialization | Returns HTTP 500 on failure instead of null body. |

## CI/CD

- **Crate publish ordering**: Fixed topological dependency order in release workflow. `ironclad-channels` moved from level 1 to level 2 (depends on `ironclad-db`), `ironclad-wallet` moved from level 2 to level 1 (depends only on `ironclad-core`). Increased crates.io index propagation wait times from 30s to 45s.
