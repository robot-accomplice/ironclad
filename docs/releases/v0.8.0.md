# v0.8.0 Release Checklist (Reliability-First)

This checklist operationalizes the roadmap selection for `v0.8.0`.

## Scope

Primary:

- `1.16` Durable Channel Delivery Queue
- `1.17` Production-Grade Abuse Protection
- `1.18` Cron-Conformant Session Rotation
- `2.14` Skills Catalog (CLI + Dashboard)

Stretch:

- `2.15` Ops Telemetry + Release Provenance Gate

## Execution Order

1. `1.16` Durable Channel Delivery Queue
2. `1.18` Cron-Conformant Session Rotation
3. `1.17` Production-Grade Abuse Protection
4. `2.14` Skills Catalog (CLI + Dashboard)
5. `2.15` Stretch (only after all primary gates are green)

## Release Gates

### 1.16 Durable Channel Delivery Queue

- [ ] Restart recovery replays pending deliveries from persistent storage.
- [ ] Idempotency prevents duplicate sends after replay.
- [ ] Dead-letter queue is inspectable and replayable via API/CLI.

### 1.18 Cron-Conformant Session Rotation

- [ ] Rotation evaluates from configured cron expression semantics.
- [ ] Edge schedule and timezone conformance tests pass.
- [ ] Migration notes are documented for behavior changes.

### 1.17 Production-Grade Abuse Protection

- [ ] Trusted proxy CIDR handling is validated.
- [ ] IP and actor (token/user) quotas are enforced and observable.
- [ ] Multi-instance limiter mode is validated.

### Legacy Loopback Proxy Removal Gate (Required)

- [ ] No runtime path auto-migrates or accepts `http://127.0.0.1:8788/<provider>` provider URLs in `v0.8.0`.
- [ ] Startup/config validation fails fast with actionable guidance when any `providers.*.url` uses the legacy loopback pattern.
- [ ] `/api/models/available` reports legacy loopback URLs as unsupported (not recoverable proxy states).
- [ ] Shipped docs/examples use direct upstream provider base URLs only.

### 2.14 Skills Catalog

- [ ] CLI list/search/install/activate workflows pass end-to-end.
- [ ] Dashboard browse/filter/multi-select/install-activate flow passes.
- [ ] Integrity verification and rollback-on-partial-failure are enforced.

### 2.15 Stretch (Conditional)

- [ ] Release pipeline `release-gates` job must pass before build/publish/release.
- [ ] Smoke gate executes critical API tests (`server_api::health_endpoint`, `server_api::session_create_and_list`).
- [ ] Provenance gate self-test passes (`scripts/generate-provenance.sh --self-test`).
- [ ] Release artifacts include `SHA256SUMS.txt` and `PROVENANCE.json`.
- [ ] Build provenance attestations are emitted for packaged binaries.

### Zero-Regression Go-Live Gate (Required)

- [ ] `just test-v080-go-live` passes before tagging or publishing.
- [ ] UAT stack passes (`scripts/run-uat-stack.sh`) including both CLI and web/dashboard smoke checks.
- [ ] Soak/fuzz battery passes (`SOAK_ROUNDS=4 FUZZ_SECONDS=45 just test-soak-fuzz`) with zero flaky retries.
- [ ] Release docs gate passes (`just test-release-doc-gate`) including changelog/release-note/install/provenance checks.

## Cut Policy

1. Cut `2.15` first.
2. If further reduction is required, trim `2.14` to CLI-first and defer dashboard catalog UI to `0.8.x`.
3. Keep `1.16`, `1.17`, and `1.18` in scope for this release profile.

## Documentation Gate (Required Before Release Close)

- [ ] Update `docs/CLI.md` for shipped CLI/operator changes.
- [ ] Update `docs/DEPLOYMENT.md` for deployment/hardening changes.
- [ ] Update `docs/architecture/ironclad-dataflow.md` for architecture/dataflow changes.
- [ ] Update `docs/ROADMAP.md` statuses to reflect shipped scope.
