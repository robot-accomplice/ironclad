# v0.8.3 Release Notes

Security hardening, auth middleware fix, and workspace coherence tooling. Closes 4 security gaps from v0.8.2 audit, adds A2A replay protection, plugin permission enforcement, and a new `ironclad defrag` command for workspace health scanning.

## Highlights

- **Auth loopback fix**: ConnectInfo now properly populated for Tower middleware, fixing the loopback auth bypass check that was silently failing (all keyless requests were rejected regardless of source IP)
- **A2A replay protection**: Nonce registry with TTL-based expiry prevents message replay attacks
- **Plugin permission enforcement**: New `strict_permissions` / `allowed_permissions` config for plugin policy
- **`ironclad defrag` command**: 6-pass workspace coherence scanner (refs, drift, artifacts, stale, identity, scripts) with `--fix` auto-repair

## Security

| Fix | Severity | Description |
|-----|----------|-------------|
| Auth bypass when no API key | Critical | Requests to non-exempt routes now fail closed when no API key is configured — only loopback connections allowed. Previously, missing ConnectInfo caused the loopback check to silently fail, rejecting all requests. |
| A2A replay protection (BUG-307) | High | Nonce registry with TTL-based expiry prevents message replay within the nonce window. |
| Plugin permission enforcement (BUG-313) | Medium | `strict_permissions` blocks undeclared plugin permissions; permissive mode (default) produces warnings. |
| EIP-191 signature recovery ID | Medium | Signatures now include recovery byte (v=27/28), producing correct 65-byte signatures. |

## Bug Fixes

| Fix | Description |
|-----|-------------|
| UTF-8 panic in memory truncation | Replaced unsafe byte-level slicing with `floor_char_boundary()` to prevent panics on multi-byte characters near the 200-char truncation point. |
| Script plugin zombie processes | Script timeout now kills and reaps child processes, preventing zombie accumulation. |
| Script plugin unbounded output | stdout/stderr capped at 10 MB via `AsyncReadExt::take()`. |
| Keystore lock ordering (BUG-300) | Consolidated two separate mutexes into single `KeystoreState` mutex, eliminating deadlock scenarios. |
| ConnectInfo for auth middleware | Changed server startup to use `into_make_service_with_connect_info::<SocketAddr>()` so peer address is available to the auth Tower middleware layer. |

## New Features

- **`ironclad defrag`**: Workspace coherence scanner with 6 passes:
  - `refs` — dead reference elimination
  - `drift` — config drift detection
  - `artifacts` — orphaned file cleanup
  - `stale` — ghost state entry removal
  - `identity` — brand consistency check
  - `scripts` — script health validation
  - Supports `--fix`, `--yes`, and `--json` flags
