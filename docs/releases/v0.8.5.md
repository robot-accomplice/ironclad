# v0.8.5 Release Notes

Security hardening and silent failure elimination. Addresses 50 findings from the v0.8.5 bug audit — 6 security fixes and 44 silent failure/correctness fixes.

## Highlights

- **WASM preemptive timeout**: Plugin execution now runs on a dedicated thread with `recv_timeout`, providing true preemptive timeout instead of post-hoc elapsed-time checks
- **Script runner orphan kill**: Child PID captured before `wait_with_output()` and killed on timeout, preventing zombie process accumulation
- **Rate limiter memory bounds**: Per-IP (10K) and per-actor (5K) map caps prevent unbounded memory growth during floods
- **Config secret allowlist**: Admin config endpoint uses field allowlist instead of blocklist for safe-by-default secret filtering

## Security

| Fix | Severity | Description |
|-----|----------|-------------|
| WASM preemptive timeout (BUG-101) | Critical | True preemptive timeout via dedicated thread, preventing indefinite execution of malicious modules. |
| Script runner orphan kill (BUG-102) | High | Explicit kill on timeout prevents orphan process accumulation. |
| Rate limiter memory bounds (BUG-103) | High | Per-IP and per-actor map caps with window-reset clearing. |
| Knowledge/Obsidian bounded reads (BUG-104, BUG-110) | Medium | 10 MB and 5 MB file size limits prevent OOM on oversized files. |
| Config secret allowlist (BUG-106) | Medium | Allowlist-based field filtering ensures new secret fields are safe by default. |
| Interview turn cap (BUG-107) | Medium | 200-turn maximum prevents unbounded memory growth within TTL. |

## Bug Fixes

| Fix | Description |
|-----|-------------|
| reqwest Client panic (BUG-105) | `VectorDbSource::new()` and `GraphSource::new()` now return `Result` instead of panicking. |
| Signal handler crash (BUG-108) | SIGTERM handler falls back to SIGINT-only mode instead of crashing in containers. |
| Heartbeat unreachable panic (BUG-109) | Catch-all arm returns safe default instead of `unreachable!()`. |
| Regex recompilation (BUG-111) | Obsidian regexes now `LazyLock` statics instead of recompiled per invocation. |
| Budget float precision (BUG-112) | Epsilon-aware comparison prevents spurious over-budget rejections. |
| Sub-agent lifecycle failures (SF-15–SF-20) | All `let _ =` patterns on registry operations now log errors. |
| API key diagnostics (SF-21, SF-22) | Missing env vars produce warn-level messages instead of silent empty strings. |
| 19 additional silent failures (SF-36–SF-52) | Error logging added across oauth, plugin-sdk, retrieval, digest, skills, signal, discord, whatsapp, sessions, defrag, embedding, main CLI, keystore, and obsidian modules. |
