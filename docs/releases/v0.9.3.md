# v0.9.3 Release Checklist (Tier-4 Wiring Completion + Channel Expansion)

This release starts immediately after `v0.9.2` wiring remediation closure.

## Scope

**Phase 0 — Deferred Wiring (from v0.9.2 Tier 4):**

- 1.17 Production-grade abuse protection
- MCP client discovery realism + protocol reachability improvements
- Cron → inference bridge
- Voice receive/transcription path wiring

**Phase 1 — Channel Expansion (from v0.9.x series):**

- 1.8 Email channel adapter hardening + thread/session behavior
- 1.4 Discord WebSocket gateway productionization
- 1.6 Multimodal message handling completion

**Phase 2 — Reliability Additions (approved scope add-ons):**

- MCP discovery cache + TTL + retry backoff controls
- Cron dry-run/simulation mode for safe schedule validation
- Integration `mechanic` checks for email/discord/voice

## Execution Order

### Phase 0: Deferred Wiring First

0a. **Abuse protection (1.17)**

- Add production abuse controls beyond baseline rate limits:
  - abuse signal aggregation (per-actor + per-origin + per-channel)
  - temporary quarantine/slowdown actions
  - operator audit trail for abuse decisions
- Ensure all entry paths use the same abuse middleware/policy checks.

0b. **MCP discovery realism**

- Replace stub discovery behavior with real remote discovery calls.
- Ensure discovered tools/resources are validated and persisted.
- Add health/error telemetry and retry behavior for discovery failures.

0c. **Cron → inference bridge**

- Enable scheduled jobs to trigger agent inference through the same unified pipeline.
- Ensure jobs carry explicit authority/channel context and audit trace.

0d. **Voice receive path**

- Wire inbound voice handling through transcription + normalization + unified request pipeline.
- Ensure parity with text channels for policy, diagnostics, and tool wiring.

### Phase 1: Channel Expansion

1a. **Email channel (1.8)**

- IMAP/SMTP/OAuth setup path and connectivity checks.
- Thread-aware session mapping and durable delivery behavior.
- Safety, deduplication, and retry semantics parity with existing channels.

1b. **Discord gateway (1.4)**

- Production gateway lifecycle (connect/reconnect/heartbeat/backoff).
- Stable dispatch of `MESSAGE_CREATE` to unified pipeline.
- Channel health metrics + operator diagnostics.

1c. **Multimodal messages (1.6)**

- Complete image/voice input handling end-to-end.
- Ensure provider translation/parsing parity for multimodal content.

### Phase 2: Reliability Additions

2a. **MCP discovery cache/TTL/backoff**

- Add discovery result caching with explicit TTL.
- Add retry backoff policy for failing MCP endpoints.
- Expose cache/refresh behavior in operator telemetry.

2b. **Cron dry-run/simulation**

- Add a dry-run path that evaluates schedules and planned inference triggers without execution.
- Provide operator-visible preview output for next run windows and inferred context.

2c. **Integration `mechanic` checks**

- Add channel readiness checks for email/discord/voice in `mechanic`.
- Ensure checks validate auth/connectivity/permissions and return operator-readable diagnostics.
