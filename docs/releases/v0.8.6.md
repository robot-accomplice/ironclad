# v0.8.6 Release Notes

Security audit remediation and test coverage expansion. Eliminates 9 security vulnerabilities (2 critical, 3 high, 4 medium) and 11 silent failure/correctness fixes from the v0.8.6 audit.

## Highlights

- **Rate-limit identity hardening**: Removed spoofable `x-user-id` header as rate-limit actor identity; IP extraction now uses real TCP peer address
- **Stable token fingerprinting**: SHA-256 replaces `DefaultHasher` for deterministic cross-process token fingerprinting
- **ASCII-only identifiers**: Closes Unicode homoglyph and normalization attacks on SQL identifiers
- **Error message sanitization**: SQLite schema-leaking prefixes added to blocklist

## Security

| Fix | Severity | Description |
|-----|----------|-------------|
| Unauthenticated rate-limit identity | Critical | Removed `x-user-id` header as rate-limit actor identity — trivially spoofable. |
| Stable token fingerprinting | Critical | SHA-256 replaces `DefaultHasher` which is not stable across processes. |
| Rate-limit IP fallback | High | IP extraction uses `ConnectInfo<SocketAddr>` instead of hardcoded `127.0.0.1`. |
| ASCII-only identifiers | High | `validate_identifier` restricts to ASCII, closing Unicode homoglyph attacks. |
| Memory search query cap | High | Query parameter capped at 512 chars to prevent regex DoS. |
| Error message sanitization | Medium | SQLite schema-leaking prefixes added to blocklist. |
| Rate-limit counter ordering | Medium | Global counter incremented after per-IP/per-actor checks pass. |
| Symlink-safe directory traversal | Medium | `collect_findings_recursive` skips symlinks. |
| WhatsApp HMAC raw byte comparison | Medium | Closes timing side-channels from hex string comparison. |

## Bug Fixes

| Fix | Description |
|-----|-------------|
| Windows daemon error propagation | `schtasks` errors propagate instead of being silently dropped. |
| CLI API key headers | All 22 bare `reqwest` calls replaced with `http_client()` helper. |
| Flaky test elimination | TOCTOU port test replaced with RFC 5737 TEST-NET-1 address. |
| Bundled providers parse failure (F5) | `.unwrap_or_default()` changed to `.expect()` for build-time data. |
| Update state save errors (F3) | `state.save().ok()` sites now log errors. |
| Auth middleware roundtrip tests | Added wrong key rejection, no-auth passthrough, POST coverage. |
| SSE streaming validation tests | Added empty/oversized/missing field coverage. |
